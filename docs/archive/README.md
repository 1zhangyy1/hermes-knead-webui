# Archive Policy

This directory is intentionally small in the public source tree.

Knead had many exploratory design notes, implementation scratchpads, and discarded plans while the product model was being formed. Those notes are not part of the release contract because they often contain obsolete product names, local development details, and directions that conflict with the current model.

Use the current documents instead:

- [`../../PRODUCT.md`](../../PRODUCT.md)
- [`../../PRODUCT_UIUX.md`](../../PRODUCT_UIUX.md)
- [`../../DESIGN.md`](../../DESIGN.md)
- [`../PRODUCT_MODEL_CONTRACT.md`](../PRODUCT_MODEL_CONTRACT.md)
- [`../architecture/PRODUCTION_REPOSITORY_PLAN.md`](../architecture/PRODUCTION_REPOSITORY_PLAN.md)

Historical notes can be kept outside the release tree when needed for internal research. They should only be promoted back into `docs/` after being rewritten as current, public-facing documentation.
