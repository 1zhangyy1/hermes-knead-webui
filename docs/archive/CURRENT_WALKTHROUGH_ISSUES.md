# 当前走查问题清单

日期：2026-05-29

本文记录当前 `vendor/hermes-webui` 版本的真实产品走查结果。判断基准以 `docs/PRODUCT_DIRECTION_CURRENT.md` 为准。

## 1. 当前状态

### 已经跑通

- Hermes WebUI 可打开：`http://127.0.0.1:8788`
- Agent 基础聊天已跑通。
- 当前默认模型已修正为 `anthropic/claude-sonnet-4.6`。
- 后端 API 和前端 UI 都能完成一轮 `只回复 OK -> OK` 的流式对话。
- 页面已经有三栏雏形：
  - 左侧：新对话 / 历史 / 工作台 / 设置
  - 中间：普通对话工作台 + Chat
  - 右侧：材料 / 结果 / 任务界面

### 尚未跑通

- “任务界面长出”还没有真实发生。
- “保存为工作台”不是从任务完成后自然出现，而是左侧已经预置了 PPT 工作台。
- “自进化”没有进入真实闭环。
- 右侧材料 / 结果仍主要是 Hermes 原本的 workspace file browser，被我们用文案包了一层。
- 工作台入口打开后，主区变成“选择一个工作台”，Chat 会消失，不符合“Chat 始终在场”。

## 2. 最大问题

### P0：产品主线仍然混乱

我们当前页面同时在讲三件事：

1. Hermes 原始 Agent WebUI：会话、文件树、技能、自动化、审批、澄清。
2. Next AI 空工作台：新对话、普通对话工作台、材料、结果。
3. 早期工作台平台：左侧工作台 tab、PPT 工作台、预览中、选择一个工作台。

用户第一眼会感觉“这到底是聊天产品，还是工作台市场，还是文件管理器”。

当前产品定义要求：

```text
新对话 -> 普通对话工作台 -> 任务中长出界面 -> 保存为专属工作台
```

但当前页面暗示的是：

```text
新对话 / 历史 / 工作台 并列
工作台里已经有 PPT 工作台
点击工作台后离开 Chat
```

这和产品定义冲突。

### P0：Chat 没有真正成为唯一主控台

在普通对话页里，Chat 是主体验；但点击左侧 `工作台` 后，主区域变成“选择一个工作台”，输入框消失。

这会把产品拉回传统 SaaS：

```text
先选工具 -> 再操作
```

而不是我们要的：

```text
先聊天做事 -> 做着做着长出工具
```

第一版需要保证任何 tab 或工作台状态下，Chat 都不能被彻底赶走。

### P0：右侧材料 / 结果不是材料 / 结果

右侧标题叫 `材料 / 结果`，但实际内容混着：

- 空工作台说明卡
- 工作区文件树
- repo 目录
- 文件大小
- 原 Hermes workspace 操作按钮

这对开发者有用，但对普通用户不是“材料 / 结果”。尤其 `.hermes-home`、`vendor`、`package.json` 这些会直接把用户拉进代码仓库语境。

MVP 0 的右侧应该先是空的任务上下文，不应该默认展示代码文件树。

### P1：工作台出现得太早

当前左侧已经有 `工作台` tab，里面直接出现 `PPT 工作台 / 预览中`。

这违背文档里的关键判断：

> 专属工作台不是用户第一步要理解的东西，而是从对话和任务中沉淀出来的结果。

第一版可以保留工作台入口，但默认应该是弱入口，甚至只显示“还没有保存的工作台”。PPT 工作台应该在用户完成一次 PPT 任务后再出现。

### P1：提案卡逻辑太机械

当前 `messages.js` 里有硬编码触发词：

```text
PPT / powerpoint / slide / deck / 路演 / 融资 / 演示文稿 / 幻灯片
```

命中后追加“建议生成 PPT 工作台”卡片。

问题：

- 它不是根据任务完成情况出现。
- 它没有和右侧任务界面、材料、结果联动。
- 它直接说“生成工作台”，会让用户提前理解工作台概念。
- 它不回答“为什么现在值得保存”。

更自然的触发时机应该是：AI 已经帮用户完成一次结构化结果后，再提示“这类任务以后可以保存为工作台”。

### P1：历史错误污染当前体验

当前会话里仍显示之前的：

```text
Error: No LLM provider configured
```

虽然现在模型已经能跑，但用户打开这个 session 时第一眼仍会以为 agent 坏了。

需要：

- 清理测试会话或归档错误会话。
- 新用户默认进入干净的新对话。
- 错误信息应该可折叠或标记为历史问题。

### P1：模型状态有潜在不一致

设置里默认模型是 `anthropic/claude-sonnet-4.6`，后端也能跑。

但空会话 DOM 里曾出现 `#modelSelect` value 为 `openai/gpt-5.4-mini`。前端发送时实际仍走 session model，所以不一定立刻坏，但这是一个隐患：

- UI 显示和 session 实际模型可能不同。
- 用户可能不知道当前到底连的什么模型。

需要统一 model selector、session model、settings default 三者。

### P2：很多 Hermes 原生能力仍在 DOM 和文案里

页面隐藏了很多高级能力，但 DOM 和可访问文本里仍有：

- 自动化
- 技能
- 记忆
- 看板
- 工作区
- 配置
- 日志
- Preview / Apply / Rollback
- New file / New folder
- Provider details

这些能力可以保留在系统里，但第一版用户界面不应该显性出现或被读屏读到。

### P2：文档状态太多，容易继续跑偏

当前有多份文档仍记录早期路线：

- `AI_SPACE_PRODUCT_MEMO.md`
- `IMPLEMENTATION_PLAN.md`
- `AGENT_WORKBENCH_PLAN.md`
- `docs/WORKBENCH_MVP_EXECUTION_PLAN.md`
- `docs/SELF_EVOLVING_WORKBENCH_CORE_PLAN.md`

虽然多数文档顶部已经标注“以 PRODUCT_DIRECTION_CURRENT 为准”，但实际执行时仍容易混用旧路线。

需要把下一阶段执行只收敛到一个短计划。

## 3. 当前不要继续做什么

先不要继续扩展：

- 工作台市场
- 多个预置工作台
- 复杂 generated space
- 插件 / 技能 / 自动化入口
- 文件树作为默认材料区
- 真正代码级自进化

这些会继续加重混乱。

## 4. 下一步建议

### Step 1：把 MVP 0 收干净

目标：打开产品后只像一个清晰可用的 AI Chat 工作台。

要做：

- 默认进入新对话或干净空会话。
- 左侧只保留：新对话 / 历史 / 工作台 / 设置。
- 工作台 tab 暂时弱化，不预置 PPT 工作台。
- 中间永远显示 Chat 和输入框。
- 右侧默认只显示材料 / 结果空状态，不显示 repo 文件树。
- 清理或隐藏历史 provider 错误。

验收：

```text
用户打开页面 -> 一眼知道可以直接输入任务 -> 发送后能收到回复
```

### Step 2：做一个假的但自然的“任务界面长出”

目标：用户输入 PPT 任务后，右侧从空状态变成 PPT 任务区。

可以先不用 AI 生成 UI，先用规则触发：

```text
用户输入包含 PPT / 路演 / slides
  -> 右侧出现：
     材料：用户目标、受众、已有素材
     结果：三页大纲
     任务界面：大纲 / 页面 / 讲稿
```

关键：Chat 仍在中间，右侧只是跟随任务长出来。

### Step 3：保存为工作台

目标：在一次 PPT 任务有结果后，再出现保存提示。

文案方向：

```text
这类 PPT 任务以后可以保存成一个工作台。
下次我会直接按“主题 -> 受众 -> 大纲 -> 页面 -> 讲稿”的流程帮你推进。

[保存为 PPT 工作台] [先不用]
```

### Step 4：再进入自进化

只有在保存后的 PPT 工作台里，才出现：

- 预览变化
- 应用
- 回滚

不要在空工作台首页讲自进化。

## 5. 当前结论

现在的状态是：

```text
Agent 已能聊天
产品壳有雏形
但信息结构仍然混杂
真正 MVP 还应该回到“极简空工作台 Chat”
```

下一轮应优先做减法，而不是继续加功能。
