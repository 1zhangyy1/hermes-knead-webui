# AI Otome

AI Otome is a single-character AI romance game example.

It shows how a Knead product can own an immersive interface while the core experience remains AI-driven: dialogue, memory, mood, affection, story beats, and choices.

## Product Shape

AI Otome demonstrates `canvas_full`: the product owns an immersive interface,
but the AI remains the product's center. The interface should support the
conversation, memory, and story state instead of becoming a static mock game.

This example is intentionally one-character first. It proves the third product
shape without introducing nested products or a separate character shelf:

- the product is the AI romance game
- the character is product data and runtime context
- chat drives relationship, mood, story beats, and choices
- generated art or save data stays out of committed source unless deliberately curated

Use this as the reference for AI-first interactive products, not as a template
for generic game engines.
